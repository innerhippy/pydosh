Options Dialog

There are two tabs that configure Bank Codes and Account Types.

Bank Codes

Some statements (eg Natwest) contain a banking code that defines the type of transaction in the statement. If this is available then it will be displayed in the main window to indicate the type of transaction. You can also then filter the records on account type - eg Direct Debits have a code 'D/D'. It doesn matter if your statements do not have this field.

To change a bank code, double click in the relevant fields and press enter.
To add a new bank code, press the  button and add the details.
You can delete a code using the button, as long as it isn't being currently used!

Press the Save button to confirm the changes.

Account Types

This section describes how pydosh interprets the CSV fields from your bank statement.

Each line in the CSV file from your bank represents a single transaction, each contains a list of fields separated by commas (hence the 'C' in 'CSV'...). Each field represents something about the transaction, the amount, date, description etc. But banks all have different ways of using the data, so pydosh needs a way of mapping each field into something it understands.

The line highlighted above represents the format for some NatWest statements - the raw CSV record will look like:

17/10/2008,C/L,"'BARCLAYS BNK 17OCT",-100.00,1234.20,"'Mr Me","'100001-12345678",

  • The date field is index 0
  • type code 1
  • description 2. 
  • As this format has negative for outgoing and positive for incoming, both the credit and debit columns are both 3 (some formats have separate fields for credit and debit).
  • balance is 4
  • currency sign says how monetary amount are repesented, supply 1 if debit is positive (money going out) or -1 if debit is negative (usual for credit statements).
  • format for the date field.
If the record has no relevant field for the index, just leave it empty.

Date Formats

Valid date formats can be built up using the following characters

  • d - the day as number (1 to 31)
  • dd - the day as number (01 to 31)
  • ddd - day name ('Mon' to 'Sun')
  • M - the month as number (1-12)
  • MM - the month as number (01-12)
  • MMM - month name ('Jan' to 'Dec')
  • MMMM - month name ('January' to 'December')
  • yy - the year as two digit number (00-99)
  • yyyy - the year as four digit number
