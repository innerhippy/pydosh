Login

This section describes how to login doshlogger and how to create a new database.

Logging into doshlogger

Doshlogger needs a database in order to store your bank transactions.

There are three options: MySql, Postgres or Sqlite. MySql and Postgres are full fat databases for which you will need admin access to configure. Sqlite is just a special file that requires no additional software or configuration.

If you choose sqlite, then just the filename is required.

If you choose MySql or Postgres you'll need to supply the correct connection details. The default ports to use are 3306 for MySql, and 5432 for postgres. For these two, you will need to create an empty database.

Creating a new database

MySql

Easiest way is to use the mysqladmin tool that ships with mysql 

mysqladmin --host localhost --user bob create doshlogger

You can call the database anything you like. You (bob) will need create privilege in order to do this. 

See here for more details.

Postgres

The admin tool command for postgres is.

createdb --host localhost --username bob doshlogger
See here for more details.

Sqlite

No initialisation is requird for sqlite - just a filename is required. The usual file extension for sqlite is ".db" but this is not enforced.

Initialising the database

doshlogger will detect an database is empty and initialise it for you. Press the Connect button and it will ask you if you want to initialise it. This assumes that the database has already been created!
